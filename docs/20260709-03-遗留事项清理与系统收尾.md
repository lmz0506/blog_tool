# 20260709-03 遗留事项清理与系统收尾

## 变更概述

集中处理历次遗留事项：草稿生成异步化、Windows 进程树终止、server 托管前端产物、清理旧产物目录、初始化 git 仓库；并按需求调整配置中心交互（默认进入配置页、tab 顺序调整）、编写 README。

| 变更 | 位置 |

## 二、遗留事项处理

### 1. 草稿生成异步化（原为同步阻塞，最长 30 分钟）

| 文件 | 变更 |
|---|---|
| `server/src/services/draftService.js` | 新增 `executeDraftGeneration`：后台调用执行器生成计划，结果/失败状态写库 |
| `server/src/routes/drafts.js` | `POST /generate` 改为创建 running 批次后立即返回 202 `{draftId}` |
| `web/src/App.jsx` | 提交后立即返回；轮询批次状态（复用现有 5 秒轮询），ready 后自动加载结果；生成页新增"后台生成中"（旋转提示）与"生成失败"（带查看日志按钮）两种状态；生成期间提交按钮禁用 |

### 2. Windows 超时杀进程不彻底

| 文件 | 变更 |
|---|---|
| `server/src/services/executorService.js` | 新增 `killProcessTree`：Windows 下用 `taskkill /PID <pid> /T /F` 整树终止（.cmd 经 shell 启动的 codex 孙进程不再残留），其他平台仍 SIGTERM |

### 3. server 托管前端构建产物

| 文件 | 变更 |
|---|---|
| `server/src/index.js` | 存在 `web/dist` 时用 `express.static` 托管——`npm run build` 后直接访问 `http://localhost:4321` 即可使用，无需 Vite |
| `web/src/api.js` | API 地址改为环境感知：Vite 开发模式指向 `http://localhost:4321/api`，构建产物走同源相对路径 `/api` |

### 4. 清理旧产物

删除空置/遗留目录：`templates/`、`storage/prompts/`、`storage/outputs/`、`storage/runs/`（旧版文件落盘方案的产物，现行数据都在数据库中）、`logs/`。

### 5. 初始化 git 版本控制

`.git` 原为空目录、无任何历史。已初始化 main 分支，添加 `.gitignore`（node_modules、web/dist、数据库文件、本地设置等），完成首次提交。

### 6. 保持现状的事项

- 执行器种子命令为本机路径：已有设置页编辑 + 自动扫描 + 文件浏览三种修正方式，且种子只在首次建库时写入，不再处理
- `node:sqlite` 实验性 API 警告：无功能影响，等 Node 转正
- API 无鉴权：本机工具，暂不处理
