# 20260710-02 安装包版本单一来源、更新自动重启与体积优化

## 问题与根因

### 1. 安装包没有显示对应的版本

- **产物版本号错误（真实 Bug）**：electron-builder 的应用目录是 `desktop/`，安装包版本取自 `desktop/package.json`。本地构建时若桌面包版本滞后于根 `package.json`，就会产出旧版本号的安装包（实例：7月10日 13:02 本地构建产出 `BlogTool-Setup-0.1.1.exe`，当时根版本已是 0.1.2）。CI 上靠人工双改 + 校验兜底，本地构建无兜底。
- **说明**：exe 的版本元数据本身一直是有的（右键 → 属性 → 详细信息可见 FileVersion/ProductVersion）；NSIS 安装向导界面默认不展示版本号，属 electron-builder 默认行为。此前应用内也无处可查当前版本。

### 2. 更新完成后不会自动打开新版本

`requestQuitAndInstallUpdate` 调用的是 `autoUpdater.quitAndInstall(false, true)`。electron-updater 中 **`isForceRunAfter` 在非静默模式（`isSilent=false`）下会被忽略**：更新走完整安装向导，装完后是否启动取决于用户在完成页的操作，不会自动重启。另外"稍后"路径（`autoInstallOnAppQuit`）是退出时静默安装，按设计不重启，属正常行为。

### 3. 打包后应用体积很大

实测安装后 357MB / 安装包 97MB，构成如下：

| 组成 | 体积 | 说明 |
|---|---|---|
| BlogTool.exe（Electron 运行时） | 216MB | 内嵌完整 Chromium + Node.js，Electron 的固有底座 |
| locales 语言包 | 47MB | Chromium 自带 50+ 种语言 |
| DLL / pak 等运行时资源 | ~45MB | 同为 Electron 固有文件 |
| 业务代码（resources/app + app-runtime） | ~9MB | 其中 node_modules 7.7MB（express 等生产依赖） |

即：**约 97% 的体积是 Electron 运行时本身**，与业务代码无关。`asar: false` 对体积几乎无影响（asar 只是打包格式不是压缩）。

## 修复内容

### 后端 / 构建配置

| 文件 | 方法/位置 | 变更 |
|---|---|---|
| `electron-builder.config.cjs` | `module.exports` | 新增 `extraMetadata.version = 根 package.json 版本`，版本号以根 package.json 为**唯一来源**，桌面包版本滞后不再影响产物；新增 `electronLanguages: ["zh-CN", "en-US"]` 裁剪语言包 |
| `scripts/electron-after-pack.cjs` | `afterPack` | 复制 `desktop/package.json` 到 `resources/app/package.json` 时，将 `version` 改写为构建元数据版本（`context.packager.appInfo.version`），保证安装后 `app.getVersion()` 与发布版本一致 |
| `desktop/main.cjs` | `requestQuitAndInstallUpdate` | `quitAndInstall(false, true)` → `quitAndInstall(true, true)`：静默安装并在完成后自动启动新版本 |
| `desktop/main.cjs` | `createTray` | 托盘悬浮提示与菜单首项显示 `BlogTool v<版本号>` |
| `desktop/main.cjs` | `createMainWindow` | 窗口标题带版本号（`砚台 · 博客自动写作台 v<版本号>`），并拦截 `page-title-updated` 防止页面 `<title>` 覆盖 |
| `desktop/package.json` | `description` | 改为中文描述（驱动安装包 exe 属性中的"文件说明"） |

### 前端

无改动。

## 验证

本地执行 `npm run dist:win` 构建验证：

- 产物名 `BlogTool-Setup-0.1.2.exe`（desktop 包版本未再参与命名）；exe 属性 FileVersion/ProductVersion = 0.1.2，文件说明为中文描述
- `win-unpacked/resources/app/package.json` 的 version 已盖章为 0.1.2
- `win-unpacked/locales/` 仅剩 `zh-CN.pak`、`en-US.pak`
- 体积：安装包 97.3MB → **89.3MB**，解压后 357MB → **311MB**
- 语法检查 `node --check` 全部通过

**更新自动重启需在下次真实发布链路中验证**；注意当前已安装的 0.1.2 仍是旧逻辑，从 0.1.2 升级到含本修复版本的那一次仍可能不自动重启，之后的升级才生效。

## 附注

- 进一步显著缩小体积需更换技术栈（如 Tauri 使用系统 WebView2，安装包可到 10~20MB），属大改造，暂不实施。
- 发布流程不变：发版仍需同时更新根与 desktop 两处 package.json 版本号（CI 校验保留），但即使漏改 desktop 侧，产物版本也不会再错。
