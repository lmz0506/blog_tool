# 20260706-02 默认任务创建修复与提示信息汉化

## 变更概述

修复"Built-in default task is disabled."报错的交互根因，并把全部服务端业务报错信息汉化；同时修复系统默认任务无知识点时写作 Prompt 空白的问题。

## 一、默认任务创建流程修复

**问题**：在配置中心「自动化」标签页勾选"启用系统内置默认任务"后，如果未点「保存自动化配置」就点「立即创建默认任务」（或任务列表页的「同步系统默认任务」），按钮的可用性判断用的是本地未保存状态，而服务端仍是关闭状态，返回 409 "Built-in default task is disabled."。

| 文件 | 方法 | 变更 |
|---|---|---|
| `web/src/App.jsx` | `handleCreateDefaultTask` | 创建/同步默认任务前先调用 `api.saveDefaultPlan(defaultPlan)` 把界面上的自动化配置同步到服务端，再创建，一步到位 |

## 二、系统默认任务 Prompt 兜底

**问题**：系统默认任务没有关联知识点（task_items 为空），写作 Prompt 中"目标知识点"为空白，Agent 缺少写作方向。

| 文件 | 方法 | 变更 |
|---|---|---|
| `server/src/services/taskService.js` | `buildTaskPrompt` | 知识点为空时改为指示 Agent：先浏览该分类目录下已有文章标题，自主选择一个有价值且未被覆盖的主题写作 |

## 三、服务端业务报错信息汉化

界面全中文但服务端报错是英文，toast 提示体验割裂。以下文件的用户可见报错全部汉化：

| 文件 | 涉及消息 |
|---|---|
| `routes/tasks.js` | 默认任务未启用（附操作指引）、默认池无可用分类（附操作指引） |
| `routes/drafts.js` | 分类名为空、分类占用中、草稿生成失败、未勾选任务项 |
| `routes/categories.js` | 分类不存在；409 判断条件从 `includes("already exists")` 同步改为 `includes("已存在")` |
| `services/taskRunnerService.js` | 任务状态不可执行、分类占用、任务执行失败、文章校验失败、文章路径缺失/越界/文件不存在 |
| `services/draftService.js` | 批次/任务项不存在、无可确认项、执行中不可删、重复确认 |
| `services/taskService.js` | 任务/知识点不存在、仅待执行可编辑、执行中不可删、默认任务执行中不可关闭 |
| `services/executorService.js` | 执行器不存在、参数模板校验、超时时间校验、命令不存在、进程失败、执行超时 |
| `services/categoryService.js` | 分类名为空、分类已存在、分类不存在 |
| `services/repositoryService.js` | 仓库路径不存在、非 git 仓库、文档目录不存在 |
| `services/gitPublishService.js` | 无文章路径不可发布 |
| `web/src/api.js` | 请求失败兜底文案 |

pino 日志（服务端控制台输出）保持英文，不影响界面。

- 实测 `POST /api/tasks/default`：当前库中默认任务开关已开启，创建成功（生成任务 #35「AI相关 / 系统默认任务」）。
