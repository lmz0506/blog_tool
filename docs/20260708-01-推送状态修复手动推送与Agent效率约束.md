# 20260708-01 推送状态修复、手动推送与 Agent 效率约束

## 变更概述

排查"页面显示发布成功但没推送到远端"的根因（网络重置导致 push 失败 + 记录逻辑 Bug），修复推送状态记录并新增页面手动推送；写作 Prompt 增加 UTF-8 强制与探索范围限制，解决日志乱码与执行过慢问题。

## 一、排查结论

### 1. 推送未生效（任务 28、29）

- `publish_result.error`：`fatal: unable to access 'https://github.com/lmz0506/blog.git': Recv failure: Connection was reset` —— **push 因网络连接被重置而失败**，commit 均已成功落在本地（仓库 ahead 2）
- 原记录逻辑 Bug：push 抛异常时整个发布结果被记为 `committed: false`，与事实不符（commit 已成功），页面信息误导
- 已手动将两条滞留 commit 推送到 origin/main（文章已上线），并修正任务 28/29 的发布记录

### 2. 执行慢（单篇约 6-10 分钟）

最新成功运行的过程日志（stderr 204KB-288KB）中出现了 doc-coauthoring 等 Claude 技能文档全文 —— **Agent 在探索仓库时阅读了 `.claude`/`.agents` 等目录下的大量无关文档**，浪费时间与 token。

### 3. 日志乱码

本批日志无 UTF-8 替换字符，乱码来自 Agent 用 Windows 命令读写文件时的本地编码（GBK）输出。

## 二、代码变更

### 1. 推送状态记录修复 + 手动推送

| 文件 | 方法 | 变更 |
|---|---|---|
| `server/src/services/gitPublishService.js` | `publishArticleToGit` | push 单独 try/catch：commit 成功如实记 `committed: true`，push 失败记 `pushed: false` + `pushError` 原因；autoPush 关闭时记 `pushSkipped` 说明。新增 `pushRepositoryBranch` 供手动推送 |
| `server/src/routes/tasks.js` | `POST /:taskId/push`（新增） | 手动推送：git push 后更新任务的 publish_result（成功清除错误标记；失败记录 pushError 并返回 502） |
| `web/src/api.js` | `pushTask`（新增） | 调用手动推送端点 |
| `web/src/App.jsx` | `handlePushTask`、`renderTaskDetail` | 发布结果面板显示"推送失败原因/推送说明"；有文章且未推送时显示「手动推送到远端」按钮 |

### 2. 写作 Prompt 约束（`server/src/services/taskService.js` `buildTaskPrompt`）

新增两条要求：
- 读写任何文件必须使用 UTF-8 编码，避免中文乱码
- 严格控制探索范围：最多只允许查看 `{docsDir}/{分类}/` 目录下的文件名和文章标题，禁止阅读 .claude、.agents、.codex、_site、node_modules、assets、admin、_layouts、_includes 等无关目录，禁止运行构建/测试等无关命令，尽快完成写作

同时明确"不要执行 git 提交或推送"（提交推送由工具负责）。

### 3. 子进程编码环境（`server/src/services/executorService.js`）

`spawnOptions`：给执行器子进程注入 `PYTHONIOENCODING=utf-8`、`PYTHONUTF8=1`、`LANG/LC_ALL=zh_CN.UTF-8`，配合 Prompt 减少乱码。

## 三、定时任务执行逻辑（现状说明）

1. server 启动时 `startScheduler`，每 60 秒检查一次（`schedulerService.tick`）
2. 前提：配置中心「自动化」→「到期任务自动执行」开关开启（`auto_schedule_enabled`）
3. 每次检查取出**全部** `status = pending` 且 `排期时间 <= 当前时间` 的任务（纯日期排期视为当天 00:00 起可执行，带时分秒的到点才执行），按排期先后串行执行（见 20260708-02）
4. 用默认执行器走完整链路：写文章 → front-matter 校验 → git 提交 → 推送（autoPush 开启时）
5. 内存 busy 锁保证同一时刻只自动执行一个任务，执行完毕后下一轮检查再取下一条；与手动执行共享任务级/分类级冲突检查
