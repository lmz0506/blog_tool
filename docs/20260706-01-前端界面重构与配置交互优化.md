# 20260706-01 前端界面重构与配置交互优化

## 变更概述

前端整体重构为「砚台 · 纸墨编辑部」设计风格，配置中心拆分为 4 个子标签页；路径类配置支持目录浏览器选择，执行器命令支持自动扫描/文件浏览/手动输入三种方式；执行器测试收进配置行内；`--add-dir` 额外目录改为自动指向工具目录，不再暴露参数模板。

## 一、UI 重构（web/src）

### 1. 设计系统（styles.css 全量重写）

- **风格**：暖纸底色 + 墨色主操作按钮 + 朱砂印章红点缀（品牌印章、激活态、焦点环），替换原蓝白 admin 模板风格
- **字体**：标题用衬线字 Noto Serif SC，正文 Noto Sans SC，路径/命令/日志用 IBM Plex Mono（`index.html` 更新字体引入）
- **组件**：统一的 entry-card（悬浮上浮 + 印章红箭头）、status-pill（带状态圆点：琥珀=进行中、黛绿=成功、朱红=失败）、toggle-row 开关行、publish-panel 发布结果面板、log-terminal 深色日志终端、empty-state 空状态、卡片入场逐个上浮动效（stagger），支持 prefers-reduced-motion
- 品牌更名为「砚台 · 博客自动写作台」（`index.html` 标题同步）

### 2. App.jsx 重构

- **页面标题区**：原先定义了但从未渲染的 PAGE_META 现在渲染为每页的衬线大标题 + 副标题，全局刷新按钮移入标题区
- **消息条改为 toast**：右上角浮出，成功消息 4 秒自动消失（`useEffect` 定时器），错误消息保留
- **配置中心拆分为 4 个标签页**（`settingsTab` state + `SETTINGS_TABS`）：
  1. 仓库与发布：仓库路径/分支/文档目录/自动推送 + 扫描 + 保存
  2. 自动化：内置默认任务开关、默认执行器、定时自动执行开关、默认池信息
  3. 分类管理：新增自定义分类 + 分类列表
  4. 执行器：执行器配置 + 行内测试
- **图标组件化**：卡片操作按钮统一为 `ActionIcon`（详情/编辑/日志/删除/重试），消除重复内联 SVG
- 删除未使用的死代码组件：LogoMark、SummaryCard、FilterBar、latestDraftRun

## 二、配置交互优化

### 1. 目录/文件浏览器（新增）

| 位置 | 说明 |
|---|---|
| `server/src/routes/system.js`（新增） | `GET /api/system/browse?path=&includeFiles=`：空路径返回 Windows 盘符列表；否则返回该目录的子目录（及可选文件）、上级目录。路径不存在/不可读返回 400 |
| `server/src/index.js` | 挂载 `/api/system`；全局错误处理支持 `error.statusCode` |
| `web/src/api.js` | 新增 `browsePath(path, includeFiles)` |
| `web/src/App.jsx` | 新增通用路径选择弹窗（`picker` state + `openPicker/loadPickerPath/pickPath`）：目录树浏览 + 手动输入路径跳转 + 目录模式「选择当前目录」/ 文件模式点击文件选中 |

应用位置：仓库路径（选择目录）、执行器工作目录（选择目录）、执行器命令（浏览文件）。所有输入框仍可手动输入。

### 2. 执行器命令自动扫描（新增）

| 位置 | 说明 |
|---|---|
| `server/src/services/executorService.js` | `discoverExecutorCommands()`：用 `where`（Windows）/`which -a` 扫描本机 PATH 中的 codex/claude/gemini/qwen/aider/cursor-agent 命令，去重返回 |
| `server/src/routes/executors.js` | 新增 `GET /api/executors/discover` |
| `web/src/App.jsx` | 执行器命令旁「自动扫描」按钮 → 弹窗列出扫描结果，点击填入 |

### 3. 额外工作目录自动化（`--add-dir` 不再暴露）

| 位置 | 说明 |
|---|---|
| `server/src/services/storage/database.js` | codex 种子参数中写死的 `E:\idea_space\blog_tool` 改为 `{toolRoot}` 占位符；新增 `migrateExecutorToolRoot` 迁移历史数据；**seedExecutors 从 ON CONFLICT DO UPDATE 改为 INSERT OR IGNORE，修复每次重启覆盖用户执行器配置的 Bug** |
| `server/src/services/executorService.js` | 运行/测试时模板上下文注入 `toolRoot`（= 工具项目根目录，来自 config.js `projectRoot`），`{toolRoot}` 运行时自动解析 |
| `web/src/App.jsx` | 执行器配置中移除「参数模板」输入框；`handleSaveExecutor` 不再提交 argsTemplate |

### 4. 执行器测试收进行内

- 删除独立的「执行器测试」面板和测试 Prompt 输入框
- 测试提示词固定在服务端（`routes/executors.js` 的 `TEST_PROMPT`），前端不可配置
- 每个执行器行操作区新增「测试」按钮：执行后 toast 提示成功/失败，输出在深色日志弹窗中查看
