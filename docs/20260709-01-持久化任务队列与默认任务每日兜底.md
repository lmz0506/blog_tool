# 20260709-01 持久化任务队列与默认任务每日兜底

## 变更概述

按既定策略重构任务执行架构：引入持久化任务队列（新任务状态 `queued` 已入队），所有执行入口（手动/调度）统一入队、单一消费者串行执行，天然去重防重复执行，程序中断重启后队列自动恢复；默认任务改为"永久存在的每日兜底任务"。

## 一、执行策略

### 默认任务（兜底）

- 默认任务**永远存在**：不存在时调度器自动从默认池随机选类补建
- 调度器每轮检查：**当天没有任何排期任务**（不论状态：待执行/已入队/执行中/已完成/已失败）**且默认任务今天还没执行过**（含手动执行，按执行记录判断）→ 默认任务当天自动执行一次
- 执行完成后的 done/failed 状态在下次触发时自动复位为待执行，保证兜底任务可反复使用

### 任务队列

- **队列即任务表**：新增任务状态 `queued`（已入队），入队 = 状态从 pending/failed 变更为 queued —— 单值状态天然去重，重复入队返回 409「该任务已在执行队列中，请勿重复添加」
- **入队来源统一**：① 调度器每 60 秒把到期（排期 ≤ 当前时间）的待执行任务入队；② 手动「立即执行」也改为入队；③ 默认任务兜底入队
- **单一消费者**（`drainQueue`）：按排期先后串行执行队列任务，同一时刻只跑一个 Agent；分类冲突（如同分类草稿在生成）的任务本轮跳过、留在队列下轮重试；其他异常任务标记 failed 离开队列，避免消费循环卡死
- **中断恢复**：队列状态持久化在数据库中；服务重启时，中断的执行中任务自动**重新入队**（原来是标记失败），已入队任务保留，启动后立即恢复消费

## 二、代码变更

| 文件 | 变更 |
|---|---|
| `server/src/services/taskQueueService.js`（新增） | `enqueueTask`（入队+去重校验）、`dequeueTask`（取消排队回到待执行）、`drainQueue`（串行消费，冲突跳过/异常出队） |
| `server/src/services/schedulerService.js` | 重写：每轮 `enqueueDueTasks`（到期任务入队）+ `maybeEnqueueDefaultTask`（每日兜底判断：`hasScheduledTasksToday` / `defaultTaskRanToday`）+ 触发消费 |
| `server/src/services/taskRunnerService.js` | `beginTaskExecution` 允许 queued 状态执行；移除 `executeTaskInBackground`（统一走队列） |
| `server/src/services/taskService.js` | 新增 `findDefaultTask`；任务列表排序加入 queued（执行中→已入队→待执行→已失败→已完成） |
| `server/src/services/maintenanceService.js` | 启动恢复：中断的 running 任务改为**重新入队**（原为标记失败） |
| `server/src/index.js` | 启动时触发 `drainQueue` 恢复消费；监听失败（端口占用）直接退出进程，不再留僵尸进程 |
| `server/src/routes/tasks.js` | `POST /:taskId/run` 改为入队+触发消费（202）；新增 `POST /:taskId/dequeue` 取消排队 |
| `web/src/App.jsx` | 新状态「已入队」：状态标/筛选项/排序/计数；详情页入队提示条 + 「取消排队」按钮；轮询条件包含 queued；执行提示改为"已加入执行队列" |
| `web/src/api.js` | 新增 `dequeueTask` |
