# 20260707-01 时间显示与排期编辑支持时分秒

## 变更概述

所有时间显示统一为完整"年-月-日 时:分:秒"（上海时区）；任务/草稿的排期时间从只能选日期升级为可编辑到时分秒，后端调度与分组逻辑同步兼容。

## 一、前端变更（web/src/App.jsx、styles.css）

### 1. 时间格式化重构

原来的 `formatDate`（只显示月/日）、`formatDateTime`（无年份无秒）、`formatFullDate`（无秒）合并重构为两个函数：

| 函数 | 用途 | 格式 |
|---|---|---|
| `formatTimestamp` | 系统时间戳（创建/更新/最近运行等，DB 存 UTC） | `2026-07-07 10:30:25`，按上海时区换算 |
| `formatSchedule` | 排期时间（用户输入的本地时间，不做时区换算） | 纯日期显示 `2026-07-07`；带时间显示 `2026-07-07 09:30:00`；空值显示"未排期" |

修正了一个隐患：排期值此前用 UTC 换算函数展示，纯日期会被错误显示成"当天 08:00"。

### 2. 排期编辑支持时分秒

4 处排期输入框从 `type="date"` 改为 `type="datetime-local" step="1"`（可编辑到秒）：生成结果行、任务项编辑弹窗、批次编辑页、任务编辑页。新增 `toDatetimeLocalValue` 兼容历史纯日期值（补 `T00:00:00` 回显）。

## 二、后端变更

| 文件 | 方法 | 变更 |
|---|---|---|
| `server/src/services/schedulerService.js` | `todayText` → `nowText`、`findDueTask` | 到期判断从"日期 <= 今天"改为"排期 <= 当前完整时间"；纯日期排期视为当天 00:00 起可执行，带时分秒的排期到点才执行（字符串比较对两种格式均成立） |
| `server/src/services/draftService.js` | `confirmDraft` | 合并任务标题中的排期展示 `T` 替换为空格 |

**排期分组语义说明**：草稿确认时仍按排期值精确分组——同一排期值的多个任务项合并为一个任务；如果给任务项设置了不同的时分秒，会各自生成独立任务（按时间点分别调度执行）。

## 三、验证

`node --check` 与 `vite build` 全部通过；全局确认无旧格式化函数残留引用。
