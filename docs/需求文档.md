# blog_tool 需求文档

> 本文档是 blog_tool 的需求总览，持续维护。每次需求变更/修复的详细内容见 docs 目录下对应的编号变更文档。

## 项目定位

blog_tool 是围绕 Jekyll 博客仓库（`E:\idea_space\blog`，线上地址 doc.limverse.com）构建的自动化写作工具：通过本机 CLI Agent（Codex / Claude Code）生成文章任务计划和文章正文，写入博客仓库 `_docs/<分类>/` 并 git 提交推送，由 GitHub Pages 自动发布上线。

## 技术架构

- **server/**：Express 4 + node:sqlite + simple-git + gray-matter，端口 4321，数据库 `storage/blog-tool.db`
- **web/**：React 18 + Vite 5 + Tailwind 3 单页应用，开发端口 4173
- **执行器**：spawn 本机 CLI Agent 子进程，Prompt 走 stdin/参数模板，从 stdout 提取带 `BLOG_TOOL_TASK_DONE` 标记的 JSON 结果

## 功能需求清单

### 1. 配置中心
- 仓库配置：博客仓库路径、目标分支、文档目录、自动推送开关
- 仓库扫描：扫描 `_docs/` 一级目录生成分类索引，解析文章 front-matter 建立文章索引；识别分类"目录名"与文章 front-matter"展示分类名"的差异（如 devops → DevOps）
- 分类管理：启用/停用、加入默认任务池、手动新增自定义分类
- 执行器管理：名称、命令、参数模板（JSON）、工作目录、超时时间、启用状态的编辑与在线测试
- 默认任务：从默认分类池随机选类的系统内置任务，开关可控
- 定时调度开关：到期任务自动执行（默认关闭）

### 2. 智能任务生成（草稿批次）
- 选择分类 + 执行器 + 数量 + 目标描述，调用 Agent 生成文章任务计划（标题 + 内容摘要）
- 未排期任务项自动按"今天起逐日递增"排期
- 草稿项可编辑（标题/摘要/顺序/排期）、单条删除、单条确认、勾选批量确认、整批确认
- 生成失败的批次状态标记 failed，可删除

### 3. 任务列表与执行
- 草稿确认后按排期日期分组生成正式任务（同日多项合并为一个任务）
- 任务执行：Agent 按 Prompt 写文章到 `_docs/<目录名>/`，front-matter 使用展示分类名
- 发布前校验：文章路径必须在文档目录内（防路径穿越）、文件必须存在；front-matter 缺失字段自动补齐（layout/title/category/date/tags）
- git 发布：文章 + assets 下新增图片一并 add/commit，自动推送可配置；发布结果（提交/推送/文件清单/错误）持久化并在任务详情展示
- 失败任务可重新执行；服务重启自动恢复卡在 running 的僵尸记录
- 定时调度：开启后服务端每分钟检查执行日期已到的待执行任务，用默认执行器自动执行

### 4. 运行记录
- 每次 Agent 调用记录 Prompt/stdout/stderr/结果到 task_runs 表，UI 可查看最近 60 条日志
