# 20260706-04 任务列表改为统一看板（筛选+分页）

## 变更概述

任务列表从"分类卡片 → 分类任务列表"两层结构改为单层看板：直接展示全部任务的分页卡片列表，按「执行中 → 待执行 → 已失败 → 已完成」+ 执行时间排序，分类与执行状态改为顶部筛选条件。

## 前端变更（web/src/App.jsx）

### 1. 页面结构

- 删除 `renderQueueCategories`（分类卡片层）与 `renderQueueTasks`（分类内任务层）两个页面，替换为统一的 `renderTaskBoard`；screen `queue-tasks` 移除，导航项 `queue-categories` 直接进入看板
- 任务详情/编辑页面包屑从「任务分类 / 任务列表 / …」简化为「任务列表 / …」
- 草稿确认、创建默认任务后跳转目标同步改为看板（自动带上对应分类和"待执行"筛选）

### 2. 排序规则（`queueTasks` memo）

1. 状态优先级：`running(执行中) → pending(待执行) → failed(已失败) → done(已完成)`（`TASK_STATUS_ORDER`）
2. 执行中/待执行组内按 `scheduledDate` 升序（越早越靠前，未排期垫底）
3. 已失败/已完成组内按 `updatedAt` 倒序（最近的在前）

### 3. 筛选条件

- **分类**：「全部」+ 从现有任务动态提取的分类列表（`taskCategoryOptions`），胶囊按钮
- **状态**：全部 / 待执行 / 执行中 / 已完成 / 已失败（`taskFilterTabs`），非"全部"选项后带实时数量（`taskStatusCounts`）
- 筛选变化自动回到第 1 页；被筛掉的分类自动重置为「全部」

### 4. 分页

- 每页 9 张卡片（`TASK_PAGE_SIZE`，3×3 网格），底部「上一页 / 第 x / y 页 · 共 n 条 / 下一页」（`.pager` 样式）；仅一页时显示总条数
- 总页数收缩时当前页自动钳制

### 5. 卡片调整

- 因跨分类展示，卡片元信息行增加分类名；状态标签移到卡片头部右上角；执行日期用短格式（MM/DD）；底部显示任务编号 #id
- 操作按钮保持不变：详情 / 编辑（待执行）/ 重新执行（已失败）/ 日志 / 删除

### 6. 样式（styles.css）

新增 `.filter-row`（筛选行）与 `.pager`（分页条）。

## 验证

`vite build` 通过；全局 grep 确认无 `queue-tasks` / `queueCategoryCards` / `taskStatusTabs` 残留引用。
